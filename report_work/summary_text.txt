P1: 摘要summary
P2: 该项目主要致力于自动理发机的设计与实现，重点解决原有结构模块化程度不足、多电机运动控制复杂的问题。团队成员完成了机械结构设计和 3D 打印原型制作，本人的工作是在该机械平台基础上设计并实现基于 ESP32 的总线舵机控制系统。系统通过串口总线控制 ST 系列总线舵机，支持位置模式、轮子模式、多电机协调运行、运动队列和传感器触发急停。
P3: 针对总线舵机常规位置模式只能做到在单圈范围内精准定位的问题，本项目通过修改总线舵机内部eeprom里的模式参数，设置成轮子模式后，实现了一种相对多圈的控制方法。该总线舵机的位置传感器分辨率为360度/4096，也就是将一圈等分为4096份，该方法通过周期性读取舵机一圈0至4095范围内的位置反馈，识别0/4095之间的跨圈跳变，将传感器的单圈位置反馈转换为累计的相对位置，再利用比例和速度反馈的思想计算速度指令，最终实现电机按照目标圈数运行并且停在目标位置附近。经过反复测试和调试，我们能够实现单电机多圈运动，位置模式和轮子模式电机混合控制，多电机顺序运行以及运动中的急停。项目仍存在断电后需重新校准、同一电机不同位置负载不同导致pid参数调整困难、软件急停不能替代硬件安全保护等限制。在未来工作中，本项目还应进一步加入硬件急停、限位保护，并在真实剪发执行机构上开展更系统的精度、安全性和可靠性测试。
P6: 3.1引言introduction
P7: 3.1.1background and motivation
P8: 随着近段时间自动化技术的普及和机器人技术的发展，越来越多传统极度依赖人工经验的服务行业开始尝试将自动化技术融入到行业内。自动理发机是一类具有明显工程挑战的人机交互机电系统。与普通工业自动化设备不同，自动理发机的工作对象是人体头部附近的头发，设备运行空间较小，安全要求极高，运动轨迹又需要具有一定的灵活性和重复性。一个能够真正工作的自动理发机不仅需要合理的机械结构，而且需要可靠的电机驱动、稳定的控制逻辑、及时的安全中断以及后续与传感器和剪发工具集成的能力。因此，即使本项目仍处于原型开发阶段，软件控制系统也必须从一开始就考虑可扩展性和安全性，而不是只能完成单次电机的转动演示。
P9: 本项目为团队设计与制作项目。队友主要负责自动理发机的机械结构部分，包括改进建模和 3D 打印制作。而本人主要负责电机驱动和多电机运行逻辑的软件部分。机械结构为电机控制提供了物理平台，但本报告的重点不是机械结构的设计，而是在已有机械平台上如何让多个总线舵机稳定、可控并且在运行过程中可以中断地运行。由于自动理发机包含旋转、升降、伸缩、靠近头部、远离头部等多个动作，单一电机的简单开环转动远远无法满足系统要求。控制系统需要能够同时处理不同工作模式的电机，并按照预设动作顺序完成复杂动作。
P10: 在本项目中，执行器我们选择采用了微雪的 ST3215-HS和SC09总线舵机，控制器采用 ESP32。选择总线舵机而不选择步进电机的原因，主要是因为总线舵机在集成度、控制便利性、多电机扩展、位置反馈以及结构紧凑性方面更符合自动理发机的设计需求。总线舵机的优点是集成度高、通信简单、可以通过 ID 控制多个电机，并且能够读取位置等反馈信息。然而，总线舵机也存在一个与本项目密切相关的限制：常规位置模式通常只能在一圈范围内进行目标位置控制，反馈值一般对应 0 至 4095 的单圈位置。当机构需要超过 360° 的连续运动时，仅依靠位置模式难以直接实现多圈目标控制。轮子模式虽然可以连续旋转，但它本质上更接近速度控制，如果没有外部软件估计转过的圈数，就无法知道电机是否已经到达目标相对位置。
P11: 3.1.2 problem statement
P12: 本项目现阶段需要解决的核心问题主要可以概括为：如何在自动理发机原型中建立一套基于 ESP32 的总线舵机软件控制框架，使其既能完成最基本的电机驱动任务，又能支持总线舵机多圈相对位置的控制、多电机之间的协调和基于传感器的急停。该问题包含三个层面。第一是驱动层面，即 ESP32 如何与总线舵机稳定通信，如何完成电机模式切换、位置读取、速度控制和停止控制。第二是逻辑层面，即如何将每个单次电机控制命令整合为可扩展的运动队列，使多个动作可以按顺序执行。第三是安全层面，即当传感器或按钮输入发生变化时，程序如何立即停止当前运动，而不是等当前动作执行完毕后才响应。
P13: 如果软件逻辑设计不合理，自动理发机的原型机就会出现多种问题。例如，轮子模式电机可能在目标位置附近无法稳定停下，或者在按钮松开后仍继续执行之前的队列动作；位置模式电机和轮子模式电机可能因为控制方式不同而难以统一管理；程序如果大量依赖 delay()这种中断指令，急停信号就可能无法被及时读取到；当多个电机共用串口总线时，如果读取和写入顺序混乱，也可能造成控制延迟或状态判断错误。因此，本项目的软件设计目标不是简单地让电机能够转起来，而是建立一套结构清晰、可调试、可扩展并具备安全中断能力的控制框架，在满足当前需求的情况下，为未来的研究打下坚实的基础。
P14: 3.3 project objectives
P15: 根据以上问题，本项目中，我给本人负责的软件部分设置了以下几项目标。第一，完成 ESP32 与总线舵机之间的通信，确保esp32开发板能够准确识别和更改舵机 ID、读取位置反馈并发送运动指令。第二，实现位置模式和轮子模式之间的切换并能完美实现这两种模式的基本驱动，使系统能够根据不同机构需求选择单圈定位或连续旋转。第三，设计相对多圈控制逻辑，通过单圈位置反馈、跨圈判断和累计位置计算，使轮子模式电机能够按照目标圈数进行相对运动。第四，建立运动队列和多电机协调逻辑，使多个动作能够连续执行，也使不同模式的电机能够在同一软件框架下运行。第五，加入安全性和可靠性措施，包括压力传感器触发、按钮暂停恢复、电流保护、堵转检测和动作超时保护。第六，通过串口调试输出提供运行状态、目标位置、累计位置、速度、电流、负载、电压和温度等信息，便于调试和后续改进。第七，通过实际联动测试验证该软件框架在单电机、多电机和急停场景下的可用性，并分析其限制和未来改进方向。
P16: 具体来说，现在我一共有5个电机，分别用M1-M5来表示。M1电机是控制底座旋转，M2电机是纵方向运动的，M3电机是控制剪刀与头距离的，M4电机是左右移动剪刀，确保剪刀能够剪到头发的，M5电机是控制剪刀开合的。以地球为例，假设地球是头，M1是控制纬度的，M2是控制经度的，M3是控制海拔的，M4和M5是精确控制的。在我的目标里，我需要完成的运行逻辑是，先保持M1电机不动，m2电机转动1/20圈；m3电机带动剪刀朝头部方向缓慢移动，在压力传感器受到压力时，立刻停下，然后往回旋转1/10圈；然后M4电机带动剪刀模块，平行地远离需要被修剪的头皮部位，然后再往回，实现头发的捕捉；最后m5电机控制剪刀剪断头发丝，然后回位；然后m3电机回到初始位置。重复这个动作，直到m2电机一共转动3圈，此时m1电机逆时针转动1/4圈，然后重复上述操作，直到m1电机一共转动7圈。此时所有动作结束，所有运动的集合能够完整覆盖头部轮廓，完成理发需求。同时软件系统带有电流检测，遇到电流较大能随时关停系统；带有启停按钮，能够手动控制系统运行，是手动的安全措施。
P17: 4. Design Approach
P18: 4.1Software design scope
P19: 本项目本人负责的范围属于自动理发机系统中的电机驱动与运行逻辑设计部分。机械结构设计、3D 打印建模与实体结构制作主要由团队其他成员完成，本项目个人贡献主要集中在电机控制系统的软件开发、运行流程设计、多电机协同逻辑、传感器响应、暂停恢复机制以及基础安全保护逻辑。软件系统的目标不是单独控制某一个电机，而是让多个执行机构按照自动理发机的工作需求形成完整的动作流程。因此，项目过程将会重点围绕如何稳定、安全、可重复地控制多个电机完成连续动作而展开。
P20: 本系统采用 ESP32 作为主控制器，通过uart通信协议，实现esp32和总线舵机驱动板之间的通信，然后使用 Serial1 串口总线与多个总线舵机通信。其中 M1 至 M4 使用 ST3215/SMS_STS 类型总线舵机，M5 使用 SC09/SCSCL 类型舵机。五个电机共用同一条串口通信总线，控制程序需要同时处理不同电机型号、不同控制模式和不同动作阶段。在顺序控制方面，状态机是一种非常适合该项目的方法。状态机把复杂流程拆分为若干清晰状态，例如等待启动、回零、初始抬升、准备M5、执行动作、M1层间移动、暂停、完成和急停。每个状态只处理特定任务，并在满足条件后转移到下一个状态。这种方法远比在loop()中堆叠大量delay()更适合自动理发机，因为这样不会中断程序，使得系统在每个控制周期都可以检查传感器、电流和堵转状态。M1、M2、M3 主要采用轮子模式进行相对位移控制，M4 和 M5 主要采用位置模式进行绝对位置控制。程序通过 FreeRTOS 多任务结构分别处理按钮输入、压力传感器输入、电机控制和串口调试信息输出，使系统在运行过程中能够持续检测外部信号并及时响应。
P21: 在反馈控制方面，M1、M2和M3采用轮子模式，由于硬件限制不能直接给定绝对多圈位置。因此程序通过读取舵机单圈位置同时处理跨圈跳变，把位置反馈变成连续的totalPos，再将targetPos与totalPos比较得到误差。控制器根据误差和速度反馈计算速度命令，使电机逐步接近目标位置。这类方法本质上是在外部控制器中实现相对位置闭环。
P23: 4.2 Design Constraints
P24: 本项目的软件设计受到机械空间、主控芯片类型、电源供电能力、通信方式、制造方式和安全需求等多方面约束。
P25: 首先，在空间约束方面，自动理发机的机械结构已经由队友完成，并且电机安装位置、运动方向和机械行程受到实际结构限制。软件设计必须适应现有机械结构，不能假设电机可以无限运动。例如，M2 作为竖直进给电机，需要先向下寻找底部位置并将底部设置为逻辑零点，然后再进行向上的分层运动。M3 作为接近动作电机，需要缓慢运动并在压力传感器触发时停止，避免继续压向目标位置。M4 和 M5 则需要在规定角度或位置之间切换，不能超过机械允许范围。
P26: 其次，在材料和制造方式方面，由于机械结构采用 3D 打印方式完成，结构刚度、连接强度和抗冲击能力有限。因此，软件控制不能采用过高速度或过大冲击式动作。程序中对电机速度、加速度、动作超时和堵转检测进行了限制。例如 M4 的位置模式设置了固定速度和加速度，M3 的接近速度被设置为较低值，以减少运动冲击。M2 在竖直运动时还加入了速度斜坡限制和重力补偿，避免电机突然加速导致机械结构受力过大。
P27: 第三，在成本和硬件资源方面，系统采用 ESP32 和总线舵机作为主要硬件平台。ESP32 成本低、体积小，并且具备多任务处理能力，适合原型系统开发。但 ESP32 与多个舵机之间共用同一串口总线，因此程序需要控制通信频率，避免频繁读写造成通信延迟或数据丢失。代码中电机控制任务每 20 ms 运行一次，传感器任务每 5 ms 检测一次，按钮任务每 10 ms 检测一次，调试任务每 1000 ms 输出一次状态信息，以平衡实时性和通信负担。
P28: 第四，在安全约束方面，自动理发机属于靠近人体运行的设备，即使当前系统仍处于原型阶段，也必须考虑停止机制和异常保护。软件中加入了按钮暂停/恢复、压力传感器触发停止、全局电流保护、单电机电流保护、堵转保护和动作超时保护。当系统检测到异常电流、堵转或超时后，会进入 emergency stop 状态，并调用 stopAllMotors() 停止所有电机，避免动作继续执行。
